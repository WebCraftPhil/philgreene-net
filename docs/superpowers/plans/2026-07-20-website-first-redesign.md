# Website-First Redesign Implementation Plan

## Goal

Reposition Phil Greene around conversion-focused local-business websites, with lead capture as the differentiator and AI reception, review requests, and retention as upgrades. The primary conversion is a Free Website Audit.

## Implementation

1. Update homepage hierarchy and copy: hero, customer-loss framing, website offer, recovery journey, automation, premium upgrades, packages, honest demonstrations, founder story, and audit.
2. Add a browser-only guided Website Checkup with accessible dialog behavior, deterministic package recommendations, and non-destructive audit-form prefilling.
3. Apply the PG dark/cyan/magenta brand system, install the optimized raster logo/favicon assets, and create the 1200×630 default social preview.
4. Update navigation, metadata, structured data, footer, and `/projects` as Work & Demonstrations.
5. Update unit and end-to-end coverage, then run lint, typecheck, tests, build, audit, and desktop/mobile visual inspection.

## Guardrails

- Keep the `/api/contact` payload schema unchanged.
- Never overwrite form values the visitor has already entered.
- Never imply client results, deployed AI, fabricated reviews, or a larger agency.
- Preserve legal routes, analytics setup, accessibility affordances, and Vercel routing.
